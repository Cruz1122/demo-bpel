grid_on Sopa de Letras: Fundamentos BPEL

Palabras a encontrar:

help Guía de Términos BPEL - Fundamentos

RECEIVE

Actividad que recibe mensajes de entrada desde servicios externos o clientes. Es el punto de entrada del proceso BPEL.

Ejemplo: Recibir una solicitud de pedido del cliente

INVOKE

Actividad que invoca servicios web externos. Puede ser síncrona (espera respuesta) o asíncrona.

Ejemplo: Llamar al servicio de pago para autorizar una transacción

REPLY

Actividad que envía respuestas de vuelta al cliente que inició el proceso. Cierra la conversación.

Ejemplo: Confirmar al cliente que su pedido fue procesado

ASSIGN

Actividad para manipular y copiar datos entre variables del proceso.

Ejemplo: Copiar información del cliente a la respuesta del pedido

VARIABLE

Almacena datos durante la ejecución del proceso. Pueden ser de diferentes tipos (mensaje, elemento XML, tipo simple).

Ejemplo: Variable que contiene los datos del pedido

SEQUENCE

Actividad estructurada que ejecuta un conjunto de actividades de forma secuencial, una tras otra.

Ejemplo: Ejecutar primero el pago, luego reservar inventario

PAYMENTRESULT

Variable específica que almacena el resultado de una operación de pago (aprobado/rechazado).

Ejemplo: Contiene si el pago fue autorizado o denegado

ORDERRESPONSE

Variable que contiene la respuesta final que se envía al cliente con el resultado del procesamiento.

Ejemplo: Mensaje de confirmación o rechazo del pedido

grid_on Sopa de Letras: Control de Flujo

Palabras a encontrar:

help Guía de Términos BPEL - Control de Flujo

IF

Estructura condicional que permite bifurcar el flujo de ejecución basado en condiciones lógicas.

Ejemplo: Si el monto > 1000, verificar crédito

CONDITION

Expresión XPath que se evalúa como verdadero o falso para determinar qué rama ejecutar.

Ejemplo: $paymentResult.approved = true

FLOW

Actividad que permite la ejecución paralela de múltiples actividades simultáneamente.

Ejemplo: Procesar pago e inventario al mismo tiempo

WHILE

Actividad de bucle que repite una secuencia de actividades mientras una condición sea verdadera.

Ejemplo: Reintentar pago mientras fallos < 3

PICK

Actividad que espera por uno de varios eventos posibles (mensajes o alarmas de tiempo).

Ejemplo: Esperar confirmación del cliente o timeout

SCOPE

Contenedor que agrupa actividades y define manejadores de errores y compensación.

Ejemplo: Agrupar operaciones de pago con manejo de errores

CREDITCHECK

Variable o proceso específico para verificar la solvencia crediticia de un cliente.

Ejemplo: Datos del cliente para verificación crediticia

APPROVED

Estado o valor booleano que indica si una operación (pago, crédito) fue aprobada.

Ejemplo: creditResult.approved = true

PARALLEL

Concepto de ejecución simultánea de múltiples actividades usando la actividad <flow>.

Ejemplo: Procesar múltiples servicios al mismo tiempo

grid_on Sopa de Letras: Servicios y Datos

Palabras a encontrar:

help Guía de Términos BPEL - Servicios y Datos

ASSIGN

Actividad fundamental para manipular datos, copiar valores entre variables y realizar transformaciones.

Ejemplo: Copiar nombre del cliente a la respuesta

COPY

Elemento dentro de <assign> que especifica una operación de copia de datos de origen a destino.

Ejemplo: <copy from="$input" to="$output">

FROM

Especifica el origen de los datos en una operación de copia. Puede ser una variable, expresión o literal.

Ejemplo: from="$customerInfo.name"

TO

Especifica el destino donde se copiarán los datos. Debe ser una ubicación escribible.

Ejemplo: to="$order.customerName"

VARIABLE

Contenedor de datos que puede almacenar mensajes, elementos XML o tipos simples durante la ejecución.

Ejemplo: Variable que contiene información del cliente

PARTNER

Referencia a un servicio externo con el que el proceso BPEL interactúa mediante PartnerLinks.

Ejemplo: Servicio de pago o inventario

ORDERDATA

Variable o estructura de datos que contiene toda la información relacionada con un pedido.

Ejemplo: Productos, cantidades, precios del pedido

CUSTOMERINFO

Estructura de datos que contiene información del cliente (nombre, dirección, contacto).

Ejemplo: Datos personales para facturación y envío

TOTALAMOUNT

Variable que almacena el monto total calculado de una transacción o pedido.

Ejemplo: Suma de productos + impuestos + envío

grid_on Sopa de Letras: Manejo de Excepciones

Palabras a encontrar:

help Guía de Términos BPEL - Manejo de Excepciones

SCOPE

Contenedor que agrupa actividades y define contexto para manejo de errores, compensación y eventos.

Ejemplo: Agrupar operaciones críticas con manejo de fallos

CATCH

Manejador que captura y procesa excepciones específicas que ocurren durante la ejecución.

Ejemplo: Capturar errores de pago para reintentarlo

FAULT

Excepción o error que puede ser lanzado durante la ejecución de actividades.

Ejemplo: Fault de servicio no disponible

THROW

Actividad que lanza explícitamente una excepción para indicar una condición de error.

Ejemplo: Lanzar error si datos son inválidos

HANDLER

Mecanismo general para manejar eventos, errores o compensaciones en un scope.

Ejemplo: faultHandler, compensationHandler

COMPENSATE

Actividad que invoca mecanismos de compensación para deshacer efectos de actividades completadas.

Ejemplo: Reembolsar pago si falla el envío

PAYMENTFAULT

Tipo específico de excepción relacionada con errores en operaciones de pago.

Ejemplo: Tarjeta rechazada, fondos insuficientes

TIMEOUT

Condición de error que ocurre cuando una operación excede el tiempo límite establecido.

Ejemplo: Servicio externo no responde en 30 segundos

ROLLBACK

Proceso de deshacer transacciones y regresar al estado anterior cuando ocurre un error.

Ejemplo: Anular todas las operaciones de un pedido fallido

grid_on Sopa de Letras: Historia y Estándares

Palabras a encontrar:

help Guía de Términos BPEL - Historia y Estándares

PROCESS

Elemento raíz que define un proceso BPEL completo. Contiene todas las actividades, variables y configuraciones.

Ejemplo: <process name="OrderProcess">

XMLNS

Declaración de espacio de nombres XML que define prefijos para elementos y tipos.

Ejemplo: xmlns:tns="http://example.com"

TARGETNAMESPACE

Espacio de nombres objetivo que identifica únicamente el proceso BPEL.

Ejemplo: targetNamespace="http://mycompany.com/bpel"

IMPORT

Elemento para importar recursos externos como WSDL, esquemas XML u otros artefactos.

Ejemplo: Importar definiciones de servicios web

WSDL

Web Services Description Language - estándar para describir servicios web y sus operaciones.

Ejemplo: Archivo PaymentService.wsdl

SCHEMA

Definición XML Schema (XSD) que especifica la estructura y tipos de datos utilizados.

Ejemplo: Esquema para validar mensajes de entrada

NAMESPACE

Identificador único (URI) que agrupa elementos relacionados y evita conflictos de nombres.

Ejemplo: http://docs.oasis-open.org/wsbpel/2.0

VERSION

Identificador de versión del estándar BPEL utilizado (1.1, 2.0) o de recursos importados.

Ejemplo: BPEL 2.0 es la versión estándar actual

grid_on Sopa de Letras: Ejecución y Motor

Palabras a encontrar:

help Guía de Términos BPEL - Ejecución y Motor

VARIABLES

Contenedor que agrupa todas las declaraciones de variables utilizadas en el proceso BPEL.

Ejemplo: <variables> contiene todas las variables

VARIABLE

Elemento individual que define una variable con su nombre y tipo de datos específico.

Ejemplo: <variable name="orderData" type="...">

MESSAGETYPE

Tipo de variable basado en un mensaje definido en WSDL. Usado para datos de entrada/salida.

Ejemplo: messageType="tns:OrderMessage"

ELEMENT

Tipo de variable basado en un elemento XML Schema. Usado para datos estructurados.

Ejemplo: element="tns:CustomerData"

TYPE

Atributo que especifica el tipo de datos de una variable (messageType, element, o tipo simple).

Ejemplo: type="xsd:string" para texto simple

CORRELATIONSET

Mecanismo para correlacionar mensajes relacionados en conversaciones asíncronas largas.

Ejemplo: Correlacionar por número de pedido

PROPERTY

Elemento de datos extraído de mensajes para usar en correlación o routing de mensajes.

Ejemplo: Extraer ID de cliente de un mensaje

ALIAS

Mapeo que especifica cómo extraer una propiedad de correlación de un mensaje específico.

Ejemplo: Cómo extraer orderID de OrderMessage

grid_on Sopa de Letras: Arquitectura SOA

Palabras a encontrar:

help Guía de Términos BPEL - Arquitectura SOA

PARTNERLINKS

Contenedor que define todas las conexiones con servicios externos que participan en el proceso.

Ejemplo: <partnerLinks> agrupa todas las conexiones

PARTNERLINK

Define una conexión específica con un servicio externo, incluyendo roles y tipos de interacción.

Ejemplo: Conexión con servicio de pago

PORTTYPE

Define un conjunto de operaciones que puede realizar un servicio web (concepto de WSDL).

Ejemplo: PaymentServicePortType con operaciones charge/refund

ROLE

Define el papel que desempeña cada participante en la interacción (cliente o proveedor).

Ejemplo: myRole="client", partnerRole="server"

SERVICE

Implementación concreta de un portType, generalmente con una URL específica y binding.

Ejemplo: PaymentService en https://pay.example.com

ENDPOINT

Punto de acceso específico (URL) donde está disponible un servicio web para invocación.

Ejemplo: https://api.payment.com/v1/charge

BINDING

Especifica el protocolo y formato de mensajes para comunicarse con un servicio (SOAP, HTTP, etc.).

Ejemplo: SOAP over HTTP binding

OPERATION

Operación específica que se puede ejecutar en un servicio web (método o función).

Ejemplo: processOrder, authorizePayment, checkInventory
